OS organization and system calls

Chapter 2 : Operating system organization An operating system must fulfill three requirements: multiplexing, isolation, and interaction 2.1 Abstracting physical resources Why do we need operating system ? One could implement the system calls as a library, then each app could even have its own library tailored to its needs. Apps could directly interact with HW resources and use those resources int the best way for apps. (OS for Embedded devices or real-time systems) The downside of this approach is, multiplexing will be troublesome.
